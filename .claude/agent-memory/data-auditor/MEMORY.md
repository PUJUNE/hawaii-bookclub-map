# data-auditor 메모리

## 대조할 3개 원본
1. Notion DB '하와이 - 인간과 기술' (원본, 멤버 전용)
2. `하와이 독서모임_책_저자_장소.xlsx` (스냅샷 — index.html보다 오래됐을 수 있음. xlsx가 뒤처진 것은 '참고' 수준)
3. `index.html`의 `M` 배열 + `CAND` 객체 (표시용 최종본)

## 의도적 예외 (오탐 주의)
- **시기(p) ≠ 실제 날짜(d)** 인 회차는 연기된 모임: n:9(22.12 시기/2023-01-07), n:10(23.01/4월),
  n:28(25.02/3월), n:29(25.04 시기/4월 27일이지만 결혼식), n:31(25.07/8월 30일). p는 노션 표기 유지가 규칙.
- **CAND n:19 == n:20**: 같은 후보 풀을 두 회차에 걸쳐 씀. 중복이지만 의도적.
- **ll 없는데 pl 있는 회차**: n:23(줌 온라인), n:29(결혼식) — 지도에 못 찍는 장소로 정상.
- **n:1 (2021)**: y:21이지만 legend에는 2021이 없음(2022–2026만 표시). 회색(#9b9992)으로만 구분 — 의도된 디자인.
- pl 필드에 서사 메모가 섞임(n:37 '석준님 두 번째 방문' 등) — 자유 텍스트라 정상.

## 과거 발견한 실제 오류 유형
- 2026-07-20: 모바일 목록 토글 버튼의 회차 수 '(39)'가 하드코딩이라 40회차 추가 때 미갱신
  → #ltCount 동적 계산으로 수정됨. **UI에 박힌 숫자 vs 실제 데이터 개수** 대조는 계속 유효한 검사 항목.
- 39.5 소수 회차가 잠시 존재했다가 40으로 정규화된 이력 — 회차 번호는 정수 연속이 규칙.

## 검사 루틴
- `M`의 n이 1..M.length 연속인지, y가 d의 연도와 일치하는지(연기 예외 감안해 p 기준으로도 확인).
- `CAND` 키가 존재하는 회차 번호인지.
- ll 형식 [lat, lng], lat 33~38.7 / lng 124~131 (대한민국 범위) 벗어나면 오류.
- xlsx 대조는 Python(openpyxl/pandas)으로 읽어 회차·제목·날짜 비교.
