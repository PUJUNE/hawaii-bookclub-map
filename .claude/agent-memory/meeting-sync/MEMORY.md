# meeting-sync 메모리

## M 배열 스키마 (index.html)
한 회차 = `{n, y, p, t, a, d, pl, ll}`
- `n`: 회차 번호. 정수, 1부터 연속. (39.5처럼 소수 회차를 쓰지 않는다 — 아래 이력 참고)
- `y`: 연도 두 자리 (21~26). 노드/배지 색상 키(YC)와 연결됨.
- `p`: 시기 문자열. 노션 DB '시기' 컬럼 그대로 (예: '25.07', '25.05–06').
- `t`: 책 제목. 테마 모임이면 테마 설명 (예: '신년 미스터리 교환회').
- `a`: 저자. 복수 저자는 쉼표+공백 구분. 저자 없는 테마 모임은 `null`.
- `d`: 한국어 날짜 문자열 (예: '2026년 7월 25일 (토)', '2023년 5월 13일 오전 11:30'). 시간 포함 가능.
- `pl`: 장소 문자열 또는 `null`(장소 기록 없음). 괄호로 부가 활동 병기 가능 (예: '왕십리 CGV').
- `ll`: `[lat, lng]` 소수 4자리. **지도에 못 찍는 장소(온라인, 결혼식 등)는 pl은 두고 ll 필드 자체를 생략** — 예: n:23 '줌 온라인 미팅', n:29 '지원님 결혼식'.

## CAND 객체
- 회차 번호 → '아쉬웠던 다른 후보작' 문자열 (노션 DB 동일 컬럼).
- 후보작이 없는 회차는 키 자체를 생략. n:19와 n:20은 의도적으로 같은 문자열(같은 후보 풀).
- 팝업 하단에 자동 표시됨.

## 파급 지점 (모임 추가 시 점검)
- `subtitle`(모임 N회 · 지도 표시 N곳)은 `M.length`/`mapped.length`로 **자동 계산** — 손대지 않는다.
- 모바일 목록 토글 버튼의 개수는 `#ltCount` span으로 **자동 계산**되도록 수정됨(2026-07-20).
  이전에는 하드코딩이라 40회차 추가 때 (39)로 남는 버그가 있었다.
- 새 **연도** 시작 시: CSS `:root`의 `--y{yy}`, JS `YC`, legend 배열(`[[22,'2022'],...]`)에 항목 추가 필요.
  노드 색이 밝은 계열이면 `YTXT`에 어두운 글자색 지정 (예: 24년 옐로 → `#3a2b00`).
- 경로 화살표는 `mapped`(ll 있는 항목) 순서대로 자동 연결 — 별도 작업 불필요.

## 이력 / 엣지 케이스
- 39.5회 '번외 모임'(왕십리 CGV 호프 관람)을 만들었다가 → 정식 40회 '커피괴담'(온다 리쿠, 2026-07-25, 왕십리 CGV)으로 승격한 이력 있음 (PR #2~#4). 번외/소수 회차는 쓰지 않기로 함.
- 시기(p)와 실제 날짜(d)의 연·월이 어긋나는 회차는 연기된 모임 (n:9, n:28, n:31 등) — 오류 아님, p는 노션 표기 유지.
- n:37 pl에는 '박노수미술관은 석준님이 오늘(2026년 7월 11일) 두 번째로 방문' 같은 서사 메모도 들어감 — 장소 필드는 자유 텍스트다.
